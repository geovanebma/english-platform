Pedagogical Engines Documentation

Pedagogical Engines of the Platform

This document brings together the cross-module engines and side support menus of English Platform. These engines are responsible for personalization, intelligent review, diagnostics, planning, retention, and the learner's global progress reading.

What This Documentation Covers

Pedagogical onboarding Global spaced review Adaptive diagnostics Automatic weekly plan Error notebook 7/30-day retention Pedagogical reports

Role of These Engines

Unlike content modules, these engines work behind the learner experience. They analyze behavior, prioritize review, detect recurring issues, build routines, and present a strategic pedagogical view of the whole learning process.

Core Engines

  • Pedagogical onboarding: maps goal, available time, level, and the ideal start mode.
  • Global spaced review: gathers fragile items from several modules into one smart queue.
  • Adaptive diagnostics: adjusts focus and next-step recommendations using real performance.
  • Weekly study plan: transforms goals and available time into an execution routine.
  • Error notebook: stores recurring personal mistakes to drive reinforcement.
  • Retention 7/30: measures what remains after the initial study window.
  • Pedagogical reports: consolidates habits, strengths, weaknesses, and recommendations.

Combined Flow

  • 1. Data collection: the learner interacts with Grammar, Flashcards, Listening, Writing, and more.
  • 2. Pedagogical reading: the engines consolidate errors, retention, skill, progress, and goals.
  • 3. Action: the platform changes the review queue, weekly plan, learner focus, and next steps.

Implementation Notes

  • Scope: centralizes the platform's technical pedagogical menus in one place.
  • Typography: uses the same local DIN Round Pro family as the product.
  • Purpose: shows how the platform becomes intelligent beyond isolated content modules.
  • Strategic value: helps present personalization and pedagogical differentiation.

Why It Matters

These engines are what turn the platform into a guided system rather than a set of disconnected tools. They increase continuity, support retention, and give the learner a more personalized path.
